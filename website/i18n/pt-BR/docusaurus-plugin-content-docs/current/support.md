# Suporte e Contato do Player IPTV

## Onde pedir ajuda

Se você precisar de assistência com o Megacubo:

- **Envie um e-mail** para **contact@megacubo.tv**
- **GitHub Discussions**: discussões técnicas e ajuda
- **GitHub Issues**: reporte bugs e solicite recursos
- **Publique perguntas** no GitHub Discussions

Verifique a documentação e o FAQ antes de contatar o suporte.

## Como pedir ajuda de forma eficaz

### Antes de contatar o suporte

1. **Verifique a documentação** para seu problema específico
2. **Pesquise issues existentes** no GitHub
3. **Tente o guia de troubleshooting** para problemas comuns
4. **Gere um relatório de diagnóstico** usando ferramentas de debug

### Ao contatar o suporte

Forneça as seguintes informações:

- **Descrição detalhada** do problema
- **Passos para reproduzir** o erro
- **Informações do sistema** (SO, versão, dispositivo)
- **Mensagens de erro** ou capturas de tela
- **O que você já tentou**
- **Logs de debug** se disponíveis (execute o script megacubo-debug)

## Relatando problemas

### Relatórios de bug

Encontrou um bug? Ajude-nos a corrigi-lo:

1. **Pesquise issues existentes** para evitar duplicatas
2. **Crie um relatório de bug detalhado** com:
   - Descrição clara do problema
   - Passos para reproduzir
   - Comportamento esperado vs real
   - Informações do sistema e logs

### Solicitações de recurso

Quer sugerir um novo recurso?

1. **Vá para a página do Megacubo no GitHub**
2. **Abra uma nova issue**
3. **Use o rótulo** `enhancement`
4. **Descreva o recurso claramente** e por que ele seria benéfico

### Modelo de relatório de issue

```markdown
**Descrição:**
[Descrição clara do problema ou recurso]

**Passos para reproduzir:**
1. [Passo 1]
2. [Passo 2]
3. [Passo 3]

**Comportamento esperado:**
[O que deveria acontecer]

**Comportamento real:**
[O que realmente acontece]

**Informações do sistema:**
- SO: [Windows/macOS/Linux/Android]
- Versão: [versão do Megacubo]
- Dispositivo: [especificações do dispositivo]

**Informações de debug:**
[Execute o script megacubo-debug e inclua os logs]

**Informações adicionais:**
[Capturas de tela, logs, etc.]
```
